kb

A filing clerk that lives in your terminal. Your AI thinks; kb does the paperwork.

A knowledge base needs two different jobs done. Thinking — read a paper, understand it, write a good summary, notice where it disagrees with another. And filing — create the right folders, preserve the raw source, stamp the log, and print exact instructions for the derived note.People abandon knowledge bases because the filing grows faster than the value. The AI is the opposite: brilliant at thinking, forgetful at filing across sessions.

kb does the deterministic filing. The AI creates the derived knowledge: the Memory and its one-line index.md catalog entry.

I.The split

The surprising part: kb has no AI inside it. When you run kb add paper.pdf, it copies the file into raw/, stamps the log, then prints a playbook — precise instructions for the agent's half.This is why kb works with any agent and installs nothing heavy. The CLI is small, dumb, reliable. The intelligence is rented from whatever agent drives it.

II.What lands on disk

One command gives you a folder of plain markdown you own.All plain text. Opens in Obsidian. Lives in git. Delete kb tomorrow and you still have everything.

For a beginner, do not open a random folder first. Run kb new research from anywhere; it creates ~/kb/research/ and records it in ~/.config/kb/config.yaml. Research is just an example name: use any simple name you want, such as books or client-notes, and replace research in the later commands and paths. Use kb init only when you want to turn the current directory into a KB.

Two folders, one rule: the agent never edits raw/. If a summary is wrong you fix the memory, never the source — so a bad pass can never corrupt your evidence.

V.The "grow later" moment

You start with nothing but files. Search is just reading the index — fine when small. When it stops being fine, kb notices and tells you why.The Advisor suggests; it never acts. You stay in control. You upgrade only when you feel the pain — and learn the reason at the exact moment it matters.

Basic Memory is a search helper that kb installs and drives for you. You do not need to install it or learn it for the beginner flow.

VI.The whole verb set

Small on purpose. You learn a handful of words; the messy bookkeeping hides behind them. That is the deep-module bargain: tiny interface, lots of behavior.

You rarely name a KB, because targeting resolves itself:The first KB you create becomes your default, so the common one-KB case needs no flag at all.

VII.You don't type any of this

In Claude Code you speak, and the agent runs the verbs and follows the playbooks:

"Add these three papers to my research KB and tell me what they agree on."

The agent runs kb add three times, follows each printed playbook to write the memories, then kb search to answer — and you watch it happen.
